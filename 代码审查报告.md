## 一、严重问题（必须修复）

### ❌ 问题 1：Android 权限不足
**文件**：`android/app/src/main/AndroidManifest.xml`

**当前状态**：
```xml
<uses-permission android:name="android.permission.INTERNET" />
```

**问题**：缺少存储权限，无法读取本地文档

**修复方案**：
```xml
<!-- 添加存储权限 -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

<!-- Android 13+ 需要更细粒度权限 -->
<uses-permission android:name="android.permission.READ_MEDIA_DOCUMENTS" />
```

**优先级**：🔴 高 - 核心功能无法使用

---

### ❌ 问题 2：项目结构混乱（重复目录）
**发现**：存在重复的 `renderer/renderer/` 和 `scripts/scripts/` 目录

**影响**：
- 构建可能使用错误文件
- 代码维护困难
- 包体积增大

**修复方案**：
```bash
# 删除重复目录
Remove-Item -Recurse -Force "renderer\renderer"
Remove-Item -Recurse -Force "scripts\scripts"
Remove-Item -Recurse -Force "main\main"
Remove-Item -Recurse -Force "preload\preload"
Remove-Item -Recurse -Force "resources\resources"
```

**优先级**：🔴 高 - 构建风险

---

### ❌ 问题 3：Android 版本配置过低
**文件**：`android/app/build.gradle`

**检查项**：
- `minSdkVersion` 是否 >= 26（Android 8.0）
- `targetSdkVersion` 是否 >= 33（Android 13）

**建议配置**：
```gradle
android {
    compileSdkVersion 34
    
    defaultConfig {
        minSdkVersion 26
        targetSdkVersion 34
        versionCode 1
        versionName "1.0.0"
    }
}
```

**优先级**：🟡 中 - 影响兼容性

### ⚠️ 问题 5：错误处理不完善
**文件**：`renderer/src/App.vue`

**问题代码**：
```javascript
const decoder = new TextDecoder('utf-8');
fileContent.value = decoder.decode(result.data);
```

**风险**：
- 非 UTF-8 编码文件会乱码
- 二进制文件可能导致崩溃

**建议**：
```javascript
// 添加编码检测和错误处理
try {
  const decoder = new TextDecoder('utf-8', { fatal: true });
  fileContent.value = decoder.decode(result.data);
} catch (e) {
  // 尝试其他编码
  const decoder = new TextDecoder('gbk');
  fileContent.value = decoder.decode(result.data);
}
```

**优先级**：🟡 中
