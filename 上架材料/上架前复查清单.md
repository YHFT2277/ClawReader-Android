## 二、性能优化检查

### 2.1 启动性能

| 指标 | 目标 | 测试方法 |
|------|------|---------|
| 冷启动时间 | < 3秒 | 清除后台后启动 |
| 热启动时间 | < 1秒 | 返回桌面再进入 |
| 启动无白屏 | 是 | 视觉检查 |

### 2.2 运行性能

- [ ] 滑动流畅，无卡顿（60fps）
- [ ] 内存占用 < 200MB
- [ ] 长时间使用不闪退
- [ ] 后台切换不丢失状态

### 2.3 包体积优化

| 项目 | 当前 | 目标 |
|------|------|------|
| APK大小 | ~30MB | < 50MB |
| 安装后大小 | ~100MB | < 150MB |

**优化建议**：
- 移除未使用的资源文件
- 压缩图片资源
- 启用代码混淆（ProGuard）
- 分离ABI（arm64-v8a / armeabi-v7a）

## 五、安全性检查

### 5.1 权限检查

| 权限 | 用途 | 是否必需 | 状态 |
|------|------|---------|------|
| READ_EXTERNAL_STORAGE | 读取文档 | 是 | ☐ |
| WRITE_EXTERNAL_STORAGE | 保存文档 | 是 | ☐ |
| INTERNET | AI功能 | 是 | ☐ |
| ACCESS_NETWORK_STATE | 网络检测 | 是 | ☐ |

**检查项**：
- [ ] 只申请必要的权限
- [ ] 权限申请时机合理（使用时申请）
- [ ] 拒绝权限后应用不崩溃
- [ ] 有权限说明提示

### 5.2 数据安全

- [ ] AI API 密钥不硬编码在代码中
- [ ] 网络请求使用 HTTPS
- [ ] 不存储敏感信息在本地
- [ ] 日志中不包含敏感信息

### 5.3 代码安全

- [ ] 启用代码混淆（ProGuard/R8）
- [ ] 移除调试日志
- [ ] 关闭调试模式
- [ ] 签名证书安全保存
